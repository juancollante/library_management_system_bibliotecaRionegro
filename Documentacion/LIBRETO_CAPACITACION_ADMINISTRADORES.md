### FASE 4: GESTIÓN DE CATÁLOGO DE LIBROS (10 minutos)

**Contenido a comunicar:**

"La gestión del catálogo es una de las funciones más importantes. Es lo primero que ven nuestros usuarios.

**Agregar un Nuevo Libro:**

1. Van a 'Gestión de Libros'
2. Hacen clic en 'Agregar Nuevo Libro'
3. Llenan el formulario con:
   - Título (obligatorio)
   - Autor/Autores
   - ISBN (si lo tiene)
   - Editorial
   - Año de publicación
   - Categoría (Ficción, Ciencia, Historia, etc.)
   - Descripción/Sinopsis
   - Número de copias (cuántas unidades hay)
   - Foto/Portada del libro
   
4. Hacen clic en 'Guardar'
5. El sistema genera un ID único para el libro

**Ejemplo práctico:**

Digamos que acabamos de recibir 3 copias de 'Cien Años de Soledad' de García Márquez:
- Título: 'Cien Años de Soledad'
- Autor: 'Gabriel García Márquez'
- Categoría: 'Ficción Latinoamericana'
- Número de copias: 3
- Descripción: 'Novela clásica que narra...'

Una vez guardado, el libro aparece en el catálogo y los usuarios pueden buscarlo.

**Editar un Libro Existente:**

1. Buscan el libro en el catálogo
2. Hacen clic en la opción 'Editar'
3. Pueden cambiar cualquier información
4. Guardan los cambios

**Casos comunes:**
- Agregaron más copias: editan el número de copias
- Eliminaron un libro dañado: reducen el número de copias
- Necesitan actualizar la foto: suben una nueva imagen

**Eliminar un Libro:**

Solo pueden hacerlo si:
- Tiene 0 copias
- No hay préstamos activos
- No hay reservas pendientes

Si intenta eliminar y hay restricciones, el sistema lo impedirá.

**Importar Libros:**

Para agregar muchos libros a la vez, pueden importar un archivo CSV:
1. Van a 'Gestión de Libros'
2. 'Importar desde CSV'
3. Descargan la plantilla
4. La llenan con los libros
5. Cargan el archivo

Esto es útil para donaciones o actualizaciones grandes."

**Acciones del Facilitador:**
- Demostrar agregar un libro paso a paso
- Mostrar la pantalla de edición
- Explicar el formato CSV

"Como administradores, pueden ver y gestionar todas las cuentas de usuarios.

**Información visible de cada usuario:**

- Nombre completo

- Estado (Activo, Inactivo, Suspendido)
- Deuda total
- Libros actuales en préstamo

**Acciones que pueden hacer:**

1. **Ver Perfil Completo**
   - Hacer clic en el usuario
   - Ver historial completo
   - Ver todos sus préstamos y reservas
   - Ver historial de multas

2. **Editar Usuario**
   - Pueden actualizar email, teléfono, etc.
   - Cambiar el rol (darle más permisos si es necesario)
   - Útil cuando alguien es promovido a bibliotecario

3. **Cambiar Estado**
   - **Activo**: El usuario puede usar el sistema normalmente
   - **Inactivo**: La cuenta existe pero no puede acceder (persona se fue)
   - **Suspendido**: Se bloquea por incumplimiento de normas (mucha deuda, comportamiento inapropiado)

4. **Resetear Contraseña**
   - Si un usuario olvidó su contraseña y no puede recuperarla
   - Ustedes generan una temporal
   - Se la envían al usuario de forma segura

5. **Eliminar Usuario**
   - Solo si la cuenta es muy antigua y se cumplen criterios
   - Genera un respaldo automático

**Casos de Uso Comunes:**

*Un usuario tiene deuda por $50.000:*
- Lo pueden suspender temporalmente
- El sistema no lo deja prestar más libros
- Pueden ver exactamente qué causó la deuda
- Una vez pague, lo reactivan

*Un usuario reporta que no recibe notificaciones:*
- Verifican su email en la cuenta
- Si es incorrecto, lo actualizan
- Pueden enviar una prueba de notificación

*Una persona fue contratada como bibliotecario:*
- Buscan su usuario
- Editan y cambiam el rol
- Ahora tiene acceso al panel administrativo"

### FASE 6: GESTIÓN DE TRANSACCIONES (8 minutos)

**Contenido a comunicar:**

"Las transacciones son los préstamos y devoluciones de libros. Aunque los usuarios pueden hacerlo por el sistema, ustedes también pueden procesarlos manualmente.

**Registrar un Préstamo Manual:**

Caso: Un usuario viene a la biblioteca y quiere un libro:

1. Van a 'Gestión de Transacciones'
2. Hacen clic en 'Nuevo Préstamo'
3. Seleccionan el usuario (buscan por nombre o ID)
4. Seleccionan el libro
5. El sistema calcula automáticamente la fecha de devolución (14 días)
6. Pueden agregar notas (ej: 'Libro en buen estado')
7. Confirman

El sistema:
- Reduce el número de copias disponibles
- Crea un registro en el historial del usuario
- Genera un recibo si lo necesitan

**Registrar una Devolución:**

El usuario devuelve el libro:

1. Van a 'Gestión de Transacciones'
2. Hacen clic en 'Nueva Devolución'
3. Buscan el usuario o el libro
4. El sistema muestra los libros activos
5. Seleccionan el que está devolviendo
6. Inspeccionan el estado del libro:
   - ¿Está en buen estado?
   - ¿Tiene daños (manchas, páginas rotas)?
   - ¿Huele bien (sin moho)?

7. Marcan el estado
8. Confirman la devolución

El sistema:
- Incrementa las copias disponibles
- Registra la devolución
- Si hay retraso, calcula la multa automáticamente
- Notifica si hay reservas pendientes para este libro

**Renovación de Préstamo:**

Un usuario puede renovar 2 veces. Si no puede por el sistema:

1. Van a la sección de préstamos del usuario
2. Hacen clic en 'Renovar' en el libro
3. Se extienden 7 días más
4. El sistema notifica al usuario

**Multas por Retraso:**

Si un libro se devuelve tarde:
- El sistema calcula automáticamente: $1.000 COP por día
- Genera un registro de deuda en la cuenta del usuario
- El usuario no puede prestar más hasta pagar

**Registro de Pago:**

Cuando el usuario paga:
1. Van a 'Multas y Pagos'
2. Buscan al usuario
3. Ven la deuda detallada
4. Hacen clic en 'Registrar Pago'
5. Especifican el monto y método (efectivo, transferencia, etc.)
6. Confirman
7. El sistema actualiza el estado

**Historial de Transacciones:**

En este apartado pueden ver:
- Todas las transacciones del sistema
- Filtrar por usuario, libro, fecha
- Ver quién procesó cada transacción
- Detectar patrones o problemas"

**Acciones del Facilitador:**
- Demostrar registro de préstamo
- Mostrar registro de devolución
- Explicar cálculo de multas
- Mostrar historial completo

   - Usuarios que han prestado libros en los últimos 30 días
   - Usuarios inactivos (señal de alerta)
   - Usuarios con mayor actividad
   - Útil para: Campañas de re-engagement

4. **Reporte de Deudas**
   - Usuarios con multas pendientes
   - Monto total por cobrar
   - Duración de la deuda
   - Útil para: Seguimiento de cobros

5. **Reporte de Transacciones**

   - Inventario completo

**Reportes Programados:**

El sistema permite programar reportes que se generan automáticamente:

### FASE 10: RESOLUCIÓN DE PROBLEMAS COMUNES (4 minutos)

**Contenido a comunicar:**

"Aquí están los problemas técnicos más comunes y sus soluciones:

**Problema: Usuario bloqueado sin razón aparente**
- Verificar si tiene deuda pendiente
- Revisar si fue suspendido manualmente
- Véase la fecha de bloqueo en el registro de auditoría
- Comunicarse con el usuario

**Problema: Libro aparece como disponible pero no lo encuentran**
- Verificar cantidad de copias en la base de datos
- Revisar si hay un error de inventario
- Hacer conteo físico de libros
- Actualizar la cantidad en el sistema

**Problema: Reporte no muestra datos correctos**
- Verificar rango de fechas del filtro
- Asegurarse que no hay errores en categorización
- Hacer clic en 'Refrescar' en el reporte
- Limpiar caché del navegador (Ctrl+Shift+Del)

**Problema: Sistema lento o no responde**
- Verificar conexión a internet
- Revisar si hay muchos usuarios simultáneamente
- Reiniciar el navegador
- Contactar al equipo técnico si persiste

**Problema: No puedo acceder al panel administrativo**
- Verificar su rol en el sistema (debe ser Admin)
- Cerrar sesión y volver a iniciar
- Probar con otro navegador
- Contactar al administrador general

**Problema: Usuario reporta que no recibe email de notificaciones**
- Verificar email en su perfil
- Pedir que revise carpeta de spam
- Enviar prueba de notificación desde el sistema
- Si sigue sin funcionar, puede deberse a configuración del servidor

**Contacto con Soporte Técnico:**
- Equipo de desarrollo: [correo técnico]
- Teléfono de emergencia: [teléfono]
- Horario: Lunes a viernes, 8am-6pm"

**Acciones del Facilitador:**
- Mencionar que es normal enfrentar problemas
- Dar confianza en resolución
- Dejar claro cuándo contactar a desarrollo
